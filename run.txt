Проект писался в Visual Studio 2019, использовалась БД postgres v10.
Для начала нужно загрузить БД. Дампы хранятся в файлах *.sql. (public - просто таблички, функции, триггеры. без записей)
Далее в appsettings.json есть строка для поключения, в моём случае 
"DefaultConnection": "Server = localhost; Database=SaleDB; Port=5432; User Id=postgres; Password=postgres"
её поменять под свои значения.
Логирование реализовано в самой БД с помощью функции и триггеров в таблице t_history.
Пункт 3.1 реализован тоже с помощью функции: если id Byer'а равен 0, то мы не пишем новые покупки в его коллекцию
Также есть Unit тесты, аля я понимаю, что это такое
Добавил поле login в Byer. Пользователь регистрируется, его логин хранится в таблице, а пароль в самом postgres(создание роли)
Хотел реализовать работу через смену "DefaultConnection", и тогда бы не пришлось мудрить с функцией проверки на пользователя.
Генерация описана в контролле GenerateDB
